关于lang属性的取值

2017-12-19HTML5techresearch

相信很多人都对于HTML中lang属性的值有一番研究，网页上也有很多相关的讨论。那么到底哪一种选择才是正确的呢？

腾讯和淘宝都是zh-CN，iphone、Mozilla和Google的developer的中文官网都是zh-CN，优酷和百度则没有添加，支付宝首页更是索性给出了en的答案。

我在百度查了下，发现在知乎上有一个相关的问答，但是杂乱无章，各方持有不同的解读，看的人一头雾水。于是我去了W3C官网，希望找到一个合适的答案。

1. 首先，W3C确定语言标签语法应该遵循IETF的BCP-47文件，它代表着一系列的RFC文件，其中，与此相关的是RFC-5646。
2. 其次，过往我们使用ISO标准去选择——ISO 639-1 two-letter language code和ISO 639-2 three-letter language code，现在W3C推荐在IANA Language Subtag Registry中查询、使用。
3. W3C还强调了：“The golden rule when creating language tags is to keep the tag as short as possible.” 也就是说，越简洁越好。

在W3C的问题板块，我找到了一个Should I use two-letter or three-letter ISO language codes in language tags?
对此W3C的回应是neither，推荐使用IANA中的命名。在另一篇文章中，明确给出了这样一句话：“Avoid region, script or other subtags except where they add useful distinguishing information. For instance, use ja for Japanese and not ja-JP, unless there is a particular reason that you need to say that this is Japanese as spoken in Japan, rather than elsewhere.” 简单来说，就是如果没有特殊原因，则不应该添加region等变量；上文中以日语举例，而以简体中文（普通话）在IANA中是这样定义的：

%%
Type: language
Subtag: cmn
Description: Mandarin Chinese
Added: 2009-07-29
Macrolanguage: zh
%%

BCP-47中的section 2.2.2还提到了：The language subtag ‘zh’ can still be used without an extended language subtag to label a resource as some unspecified variety of Chinese, while the primary language subtag (‘gan’, ‘yue’, ‘cmn’) is preferred to using the extended language form (“zh-gan”, “zh-yue”, “zh-cmn”).

所以，简体中文使用cmn是最好的选择。

另有几点需要注意：

  • At all times, language tags and their subtags, including private use and extensions, are to be treated as case insensitive: there exist conventions for the capitalization of some of the subtags, but these MUST NOT be taken to carry meaning.
    即不区分大小写。
  • 强烈建议各位去查阅英文原文档，从而对于lang的取值有一个切身的体会。相信大家会好好利用CTRL+F这个快捷键和你的翻译工具。

关于简繁体的问题，如果你需要参考的话：

%%
Type: redundant
Tag: zh-cmn-Hans
Description: Mandarin Chinese (Simplified)
Added: 2005-07-15
Deprecated: 2009-07-29
Preferred-Value: cmn-Hans
%%
Type: redundant
Tag: zh-cmn-Hant
Description: Mandarin Chinese (Traditional)
Added: 2005-07-15
Deprecated: 2009-07-29
Preferred-Value: cmn-Hant
%%

当然，推广一致的规范一直是世界性的难题，希望各位都能贡献出自己的一份力量。
